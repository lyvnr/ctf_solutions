# The `inhere` directory contains 20 directories. 
# instead of traversing them one by one, 
# it's possible to find the desired file with a single command.

bandit5@bandit:~/inhere$ find . -type f -size 1033c ! -executable
./maybehere07/.file2
bandit5@bandit:~/inhere$ cat ./maybehere07/.file2
HWasnPhtq9AVKe0dmk45nxy20cvUa6EG

!!! Command explanation:
`find .`: Starts the search from the current directory (.). It includes all subfolders.

`-type f`: Search only for files (do not include folders, symbolic links, etc.).

`-size 1033c`: Find files with a size of exactly 1033 bytes. The 'c' stands for 'character', meaning byte.

`! -executable`: Find files that are not executable. The exclamation mark (!) at the beginning is a logical operator meaning "otherwise".

[+]Level 6 -> Level 7:
# The password for the next level is stored somewhere on the server and has all of the following properties:
# owned by user bandit7, owned by group bandit6, 33 bytes in size

bandit6@bandit:~$ find / -user bandit7 -group bandit6 2>/dev/null
/var/lib/dpkg/info/bandit7.password
bandit6@bandit:~$ cat /var/lib/dpkg/info/bandit7.password 
morbNTDkSW6jIlUc0ymOdMaLnOlFVAaj

!!! Command explanation:
`find /`: Starts the search from the root directory (/). This scans the entire system and all folders.

`-user bandit7`: Filters files owned by `bandit7`.

`-group bandit6`: Filters files in the group `bandit6`.

`/dev/null`: This part is one of the most critical aspects of the command.

Because you're scanning the entire system, you'll get "Permission denied" errors for folders you don't have access to.

`2` represents the error stream (stderr).

`/dev/null` sends these errors to an empty file known as a "black hole".

Result: You'll only see files that were successfully found, eliminating the clutter of errors.

!!! Command explanation: 
echo "...": Passes the seemingly complex text inside the quotation marks to the next command.

| (Pipe): Passes the output on the left as input to the command on the right.

base64 -d: Decodes the incoming data from Base64 format (-d = decode).
